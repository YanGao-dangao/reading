## 2. 一句话需求

> 提供一个 Web 平台，用户输入书名或上传图书文件，系统获取/解析内容后展示大纲，并可按章节生成缩略、详细内容或自定义风格的知识心得，帮助用户快速完成知识学习。

---

## 3. 背景与动机

| 维度     | 说明                                                                                                       |
| -------- | ---------------------------------------------------------------------------------------------------------- |
| 痛点     | 阅读一本新书（技术/管理/哲学/育儿）前缺乏快速全貌预览手段；逐章精读成本高；笔记整理耗时且风格难以统一       |
| 现有方案 | 手动搜索书评/大纲，逐章阅读后手写笔记，效率低、信息分散                                                   |
| 目标     | 通过 AI 能力，将「获取大纲 → 章节内容摘要 → 知识心得输出」流程自动化，把学习准备时间从小时级缩短到分钟级     |

---

## 4. 目标用户

| 角色           | 典型画像                                                           |
| -------------- | ------------------------------------------------------------------ |
| 主要用户       | 需要快速学习新书内容的知识工作者（程序员、管理者、新手父母等）       |
| 团队/社区用户  | 读书会、学习小组成员，需共享同一本书的学习产出                       |

---

## 5. 核心使用场景

| 编号 | 场景                         | 描述                                                       |
| ---- | ---------------------------- | ---------------------------------------------------------- |
| S1   | 新书快速了解全貌             | 输入书名 → 系统搜索网络资源 → 展示大纲 → 快速浏览章节结构   |
| S2   | 针对特定章节做深度学习       | 选择一个或多个章节 → 查看缩略/详细内容 → 标记重点           |
| S3   | 输出个性化知识心得           | 选择章节 → 自定义风格/字数 → 生成知识心得 → 复制或导出       |
| S4   | 复习已读书籍的核心知识       | 从历史记录中找到书籍 → 快速回顾大纲和之前生成的心得         |

---

## 6. 主路径（MVP）

```
用户输入书名
  → 系统搜索网络资源，获取图书内容
  → 展示大纲目录
  → 用户选择章节（单选/多选）
  → 选择输出模式：缩略内容 / 详细内容 / 知识心得
    → 若「知识心得」：选择语气风格 + 设定字数（默认 300 字）
  → 系统调用 LLM 生成内容
  → 用户查看、调整参数、重新生成
  → 内容持久化保存
```

---

## 7. 首期范围与优先级

| 优先级 | 功能域             | 说明                                      |
| ------ | ------------------ | ----------------------------------------- |
| P0     | 网络资源搜索       | 输入书名，搜索网络获取图书内容             |
| P0     | 大纲预览           | AI 根据获取的内容生成大纲目录               |
| P0     | 章节内容输出       | 支持缩略/详细两种模式                      |
| P0     | 知识心得输出       | 支持自定义语气风格和字数                    |
| P0     | 数据持久化         | 书籍、大纲、生成内容需保存                  |
| P1     | 文件上传解析       | 支持 PDF/EPUB/TXT/Markdown 上传及目录提取   |
| P1     | 用户账号体系       | 多用户隔离、团队共享                        |

## 8. 关键约束

| 约束         | 说明                                                     |
| ------------ | -------------------------------------------------------- |
| AI 模型      | 优先使用 DeepSeek，需支持模型切换                         |
