各阶段详解

🎯
Phase 0 · 触发与 P0 澄清
作用：把用户模糊的一句话想法，拆解为结构化的需求输入。AI 不急于给方案，而是先问清楚五个维度：
  • 用户与场景 — 谁在什么设备、什么时机下使用？
  • 对象与范围 — 输入格式、数据来源、品类、是否持久化？
  • 主路径 — 端到端操作流是否理解正确？
  • 规则与边界 — LLM 选型、风格定义、字数、MVP 边界？
  • 验收口径 — 怎样算「做完了」、性能预期、优先级？
为什么不跳过：后续所有分析和方案都建立在这些输入上，澄清不充分 = 方案跑偏。
product-manager P0 澄清问题清单
🔬
Phase 1 · 分析阶段（Step 1–8）
作用：从产品核心到方案决策的完整推导链，确保「方案有依据、决策有对比、风险有预判」。
  • Step 1-3（建立认知）：核心价值 → 成功标准 → 用户故事，形成分析底座
  • Step 4（关卡）：反证必要性 — 不做的影响？替代方案？若不必要则停止，不浪费后续步骤
  • Step 5：在约束内拆解实现维度，形成方案空间
  • Step 6（专家评审）：调用 product-expert 做独立评审，弥补主分析的盲点（逻辑漏洞、遗漏边缘场景、可扩展性）
  • Step 7：基于专家建议输出 2-3 个方案，每方案配 Mermaid 流程图 + 对比表，附推荐倾向和反思 pass
  • Step 8（关卡）：最终方案 + 假设 + 本期不做，用户必须确认后才进入执行
product-manager product-expert 方案对比表 Mermaid 流程图
📦
Phase 2 · 执行阶段（Step 9–10）
作用：将确认方案转化为研发可执行的标准文档 + 可点击原型。
  • Step 9 文档输出：
      • 01-需求卡：一句话需求 + 背景 + 用户 + 主路径 + 优先级（高层全局视角）
      • 02-需求方案：架构图 + 模块设计 + 技术选型 + 页面结构 + 流程图（研发落地视角）
      • 03-功能清单与验收标准：功能项（F01-FNN）+ 验收用例（TC）+ 自检表 + 关联依赖
        03 的质量由八步防遗漏流程保证：
        ① 方案文档扫描 → ② 流程图节点映射 → ③ 隐性功能检查 → ④ 原型对照
        → ⑤ CRUD 与主对象过一遍 → ⑥ 关联需求梳理 → ⑦ 必检项自检 → ⑧ product-expert 专项审查
  • Step 10 可交互原型：
      • 读取 pm-prototype 技能，按工程规范生成单文件 HTML
      • 唯一依据是 03，逐条实现 P0 功能，P1 做占位
      • 从 reference.md 选设计系统（Dark Academic / Clean Light / Warm Neutral）
      • 交付时更新 03 中的「原型实现情况对照表」
product-manager product-expert（03审查） pm-prototype 01-需求卡.md 02-需求方案.md 03-功能清单.md prototype/index.html
✅
Phase 3 · 验收阶段（Step 11–13）
作用：自动化验证原型是否覆盖所有 P0 功能，失败项形成问题表交给研发，形成闭环。
  • Step 11：生成 Playwright 验收脚本（TC 与 03 一一对应）+ README 运行说明
  • Step 12：前置检查（环境 / 入口 / 映射 / 证据路径 / 范围）→ 用户确认 → 执行 → 收集结果
  • Step 13：
      全通过 → 验收通过摘要
      有失败 → 验收问题表（TC ID + 功能项 + 预期/实际 + 严重程度 + 复现步骤）→ 交研发修复 → 回归验收
acceptance.spec.mjs 验收 README.md 验收问题表.md

Skills 技能总览

product-manager
路径：.cursor/pm-flow-skills/product-manager/
覆盖阶段：Phase 0 → Phase 2（全程主控）
核心能力：P0 澄清 → Step 1-8 分析链 → 方案文档模板 → 功能清单八步防遗漏 → 输出字数控制 → 正反例约束 → 检查清单
product-expert
路径：.cursor/pm-flow-skills/product-expert/
覆盖阶段：Step 6（方案评审）+ Step 9（03 功能清单专项审查）
核心能力：独立评审视角，检查流程合规 / 产出完整性 / 逻辑一致性 / CRUD 遗漏 / TC 映射。只输出问题不输出通过项，每条 ≤30 字，总输出 ≤400 字
pm-prototype
路径：.cursor/pm-flow-skills/pm-prototype/
覆盖阶段：Step 10（可交互原型）
核心能力：单文件 HTML 工程规范 → 三套设计系统（Dark Academic / Clean Light / Warm Neutral）→ 屏幕切换系统 → 6 种通用组件模板（Toast / Skeleton / 空状态 / Modal / 侧栏 / Tab）→ Mock 数据规范 → 流式输出 → 界面效果自检清单
