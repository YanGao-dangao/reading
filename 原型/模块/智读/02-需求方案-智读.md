# 02-需求方案：智读 — 图书知识快学平台

## 1. 方案概述

智读是一款 Web 应用，核心价值是让用户通过输入书名快速获取图书大纲，并可按章节生成不同粒度的内容摘要和个性化知识心得，实现「分钟级」图书知识学习。

---

┌──────────────────────▼──────────────────────────┐

## 3. 核心模块设计

### 3.1 网络资源搜索模块

| 项目     | 方案                                                                                             |
| -------- | ------------------------------------------------------------------------------------------------ |
| 触发     | 用户输入书名后点击"搜索"                                                                         |
| 搜索策略 | 优先调用搜索引擎 API（如 SerpAPI / Tavily / DuckDuckGo）获取该书公开内容摘要、目录、书评等        |
| 内容聚合 | 将多源搜索结果去重、清洗后，交给 AI 模块进行结构化整理                                            |
| 缓存     | 同一书名的搜索结果缓存，避免重复调用                                                              |
| 降级     | 搜索引擎无结果时，由 AI 基于自身知识生成图书概要（标注"AI 生成，非原始内容"）                      |

### 3.2 大纲生成模块

| 项目       | 方案                                                                      |
| ---------- | ------------------------------------------------------------------------- |
| 网络资源   | AI 基于搜索到的内容生成结构化大纲（章 → 节层级）                            |
| 文件上传   | （P1）从 PDF/EPUB 解析 TOC；无 TOC 时由 AI 根据全文内容生成                 |
| 大纲格式   | JSON 树结构存储，前端渲染为可折叠目录树                                     |
| 持久化     | 大纲生成后保存到数据库，用户可多次查看                                      |

### 3.3 章节内容输出模块

| 项目       | 方案                                                                              |
| ---------- | --------------------------------------------------------------------------------- |
| 输入       | 用户在大纲中选择一个或多个章节                                                      |
| 缩略模式   | AI 生成该章节核心要点摘要（默认 200-300 字），突出关键概念和结论                     |
| 详细模式   | AI 生成该章节详细展开内容（默认 800-1500 字），涵盖论点、论据、案例                  |
| 多章节     | 多选时逐章生成，合并为一个完整输出                                                  |
| 重新生成   | 用户可点击"重新生成"获取不同版本                                                    |

### 3.4 知识心得输出模块

| 项目         | 方案                                                                          |
| ------------ | ----------------------------------------------------------------------------- |
| 输入         | 用户选择章节 + 选择语气风格 + 设定字数                                          |
| 语气风格选项 | 预设：学术严谨 / 通俗易懂 / 幽默轻松 / 批判思辨 / 实战导向；支持用户自定义输入   |
| 字数控制     | 默认 300 字，用户可在 100-3000 范围内自定义输入                                  |
| 输出格式     | 默认笔记要点体（带标题 + 编号列表），结构清晰易读                                |
| 调整重生成   | 用户可修改风格/字数后重新生成                                                    |

### 3.5 数据持久化模块

| 项目       | 方案                                                          |
| ---------- | ------------------------------------------------------------- |
| 存储内容   | 书籍元信息（书名、来源）、大纲结构、生成内容（含参数快照）     |
| 历史记录   | 用户可查看历史搜索/生成的所有书籍，快速回溯                    |
| 存储方案   | MVP 使用 SQLite（单体部署简单），后续可迁移 PostgreSQL          |

## 4. 技术选型（建议）

| 层         | 技术                                             | 理由                                      |
| ---------- | ------------------------------------------------ | ----------------------------------------- |
| 前端       | Next.js（React）+ Tailwind CSS                   | SSR 友好、响应式开发高效、生态成熟          |
| 后端       | Next.js API Routes / Node.js                     | 与前端同构，降低部署复杂度                  |
| AI 模型    | DeepSeek API（优先）                              | 用户指定偏好，性价比高                      |
| 搜索       | Tavily Search API / SerpAPI                       | 结构化搜索结果，API 调用简单               |
| 数据库     | SQLite（MVP）→ PostgreSQL（扩展期）               | MVP 阶段零运维，后续可平滑迁移              |
| 部署       | Vercel / Docker                                   | 快速上线、易于扩展                          |

## 5. 页面结构

```
首页（搜索入口）
  ├── 搜索框 + 搜索按钮
  ├── 历史书籍列表（快速访问）
  └── 搜索结果 → 点击进入书籍详情

书籍详情页
  ├── 左侧/顶部：大纲目录树（可折叠、可多选）
  ├── 右侧/主区域：内容展示区
  │     ├── 模式切换 Tab：缩略 / 详细 / 知识心得
  │     ├── 知识心得参数面板（风格选择 + 字数输入）
  │     ├── 生成内容展示
  │     └── 操作栏：重新生成 / 复制 / 导出
  └── 底部：加载状态 / 生成进度
```

## 7. 异常与边界处理

| 场景                     | 处理方式                                                    |
| ------------------------ | ----------------------------------------------------------- |
| 搜索无结果               | 提示用户该书未找到网络资源，由 AI 基于自身知识生成（标注来源） |
| AI 生成超时              | 前端展示加载动画 + 超时提示，支持重试                         |
| AI 生成内容质量差        | 提供"重新生成"按钮，支持用户反馈                              |
| 网络断开                 | 前端离线提示，已缓存内容仍可查看                              |
| 多章节内容过长           | 分章节流式输出，避免一次性等待过久                             |
| 书名模糊/多个同名书籍    | 搜索结果列表展示，用户选择正确的版本                           |

## 8. 非功能需求

| 维度     | 要求                                                 |
| -------- | ---------------------------------------------------- |
| 响应式   | 桌面端（≥1024px）和移动端（≤768px）均可良好使用       |
| 可用性   | 主流程操作步骤 ≤ 4 步即可获得生成内容                  |
| 可扩展   | 模型层抽象，后续可切换 GPT/Claude 等其他模型           |
| 数据安全 | 用户生成数据隔离存储，不对外泄露                       |
