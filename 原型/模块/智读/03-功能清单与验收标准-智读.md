# 03-功能清单与验收标准：智读 — 图书知识快学平台

## 1. 功能清单

> **说明**：优先级 P0 = 首期必须交付，P1 = 后续迭代。首期聚焦网络资源搜索路径，文件上传为 P1。

| 序号   | 功能域         | 功能名称                     | 功能描述                                                                                   | 优先级 |
| ------ | -------------- | ---------------------------- | ------------------------------------------------------------------------------------------ | ------ |
| F01    | 书籍搜索       | 书名输入与搜索               | 用户在搜索框输入书名，点击搜索或回车触发网络搜索                                             | P0     |
| F02    | 书籍搜索       | 搜索结果展示                 | 展示搜索到的书籍列表（书名、作者、简介），支持用户选择正确版本                                 | P0     |
| F03    | 书籍搜索       | 搜索无结果降级               | 搜索无结果时提示用户，并由 AI 基于自身知识生成概要（标注"AI 生成"来源标签）                   | P0     |
| F04    | 大纲预览       | 大纲生成                     | 搜索/选择书籍后，AI 根据获取的内容自动生成章 → 节层级的结构化大纲                             | P0     |
| F05    | 大纲预览       | 大纲目录树展示               | 大纲以可折叠/展开的目录树形式展示，支持点击跳转                                               | P0     |
| F06    | 大纲预览       | 大纲持久化                   | 生成的大纲保存到数据库，再次访问同一书籍时直接加载                                             | P0     |
| F07    | 章节内容输出   | 单章节选择                   | 用户在大纲中点击选择单个章节                                                                 | P0     |
| F08    | 章节内容输出   | 多章节选择                   | 用户在大纲中通过复选框选择多个章节                                                            | P0     |
| F09    | 章节内容输出   | 缩略内容生成                 | 选择"缩略"模式后，AI 生成该章节核心要点摘要（约 200-300 字），突出关键概念和结论               | P0     |
| F10    | 章节内容输出   | 详细内容生成                 | 选择"详细"模式后，AI 生成该章节详细展开内容（约 800-1500 字），涵盖论点、论据、案例             | P0     |
| F11    | 章节内容输出   | 多章节合并输出               | 选择多个章节时，逐章生成内容，合并为一份完整输出，各章以标题分隔                               | P0     |
| F12    | 章节内容输出   | 内容重新生成                 | 用户可点击"重新生成"按钮获取同章节的不同版本内容                                               | P0     |
| F13    | 知识心得       | 知识心得入口                 | 在输出模式 Tab 中提供"知识心得"选项，选择后展开参数面板                                        | P0     |
| F14    | 知识心得       | 语气风格选择                 | 提供预设风格：学术严谨 / 通俗易懂 / 幽默轻松 / 批判思辨 / 实战导向，同时支持自定义输入         | P0     |
| F15    | 知识心得       | 字数设定                     | 提供字数输入框，默认 300 字，范围 100-3000 字，带输入校验                                      | P0     |
| F16    | 知识心得       | 心得生成                     | 根据用户选择的章节、风格、字数，AI 生成笔记要点体的知识心得                                    | P0     |
| F17    | 知识心得       | 心得调整重生成               | 用户修改风格或字数后，可重新生成心得内容                                                       | P0     |
| F18    | 内容交互       | 内容复制                     | 生成内容区域提供"一键复制"按钮，复制 Markdown 格式内容到剪贴板                                  | P0     |
| F19    | 内容交互       | 流式输出                     | AI 生成内容时流式展示（打字机效果），降低用户等待焦虑                                           | P0     |
| F20    | 内容交互       | 加载状态                     | 搜索和生成过程中展示加载动画/骨架屏，超时后提示重试                                            | P0     |
| F21    | 数据持久化     | 书籍历史列表                 | 首页展示用户历史搜索/生成过的书籍列表，支持点击快速回访                                         | P0     |
| F22    | 数据持久化     | 生成内容保存                 | 每次生成的内容（含输出模式、参数快照）自动保存到数据库                                          | P0     |
| F23    | 数据持久化     | 历史内容回看                 | 进入书籍详情页时，可查看该章节之前生成过的历史内容                                              | P0     |
| F24    | 响应式布局     | 桌面端布局                   | ≥1024px 宽屏下，左侧大纲 + 右侧内容的双栏布局                                                | P0     |
| F25    | 响应式布局     | 移动端布局                   | ≤768px 窄屏下，大纲和内容上下排列或抽屉切换                                                   | P0     |
| F26    | 文件上传       | 文件上传入口                 | 支持上传 PDF/EPUB/TXT/Markdown 文件                                                          | P1     |
| F27    | 文件上传       | 文件解析与目录提取           | 解析上传文件内容，自动提取 TOC；无 TOC 时由 AI 生成大纲                                        | P1     |
| F28    | 用户体系       | 用户注册/登录                | 支持多用户账号隔离                                                                            | P1     |
| F29    | 用户体系       | 团队共享                     | 同一团队/社区成员可共享书籍学习产出                                                            | P1     |

## 2. 验收标准

### 2.1 书籍搜索

**TC-READ-001：书名输入与搜索 — 正常搜索**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 用户已打开首页                                            |
| 步骤 | 1. 在搜索框输入书名（如"深入理解计算机系统"）<br>2. 点击搜索按钮或回车 |
| 期望 | 展示搜索结果列表，包含书名、作者、简介；列表非空            |

**TC-READ-002：搜索结果选择**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 搜索结果已展示                                            |
| 步骤 | 1. 点击搜索结果中的一本书                                  |
| 期望 | 跳转到书籍详情页，系统开始生成大纲                          |

**TC-READ-003：搜索无结果降级**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 用户已打开首页                                            |
| 步骤 | 1. 输入一个极其冷门的书名（如"zzz_不存在的书_zzz"）<br>2. 点击搜索 |
| 期望 | 提示"未找到网络资源"，并提供 AI 基于知识生成的概要，内容带"AI 生成"标签 |

**TC-READ-004：空搜索校验**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 用户已打开首页                                            |
| 步骤 | 1. 搜索框为空时点击搜索                                    |
| 期望 | 提示"请输入书名"，不发起搜索请求                            |

### 2.2 大纲预览

**TC-READ-005：大纲自动生成**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 用户已选择一本搜索到的书籍                                 |
| 步骤 | 1. 进入书籍详情页                                         |
| 期望 | 自动生成并展示大纲目录树，至少包含章级别结构                 |

**TC-READ-006：大纲折叠/展开**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 大纲已生成展示                                            |
| 步骤 | 1. 点击有子节点的章标题旁的折叠/展开图标                    |
| 期望 | 子节点正确折叠/展开，动画流畅                               |

**TC-READ-007：大纲持久化与再加载**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 大纲已生成                                                |
| 步骤 | 1. 返回首页<br>2. 从历史列表点击该书再次进入详情页           |
| 期望 | 大纲直接加载展示，无需重新生成                               |

### 2.3 章节内容输出

**TC-READ-008：单章节缩略内容**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 大纲已展示                                                |
| 步骤 | 1. 点击选择一个章节<br>2. 选择"缩略"模式<br>3. 等待生成     |
| 期望 | 流式输出该章节核心要点摘要，字数约 200-300 字               |

**TC-READ-009：单章节详细内容**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 大纲已展示                                                |
| 步骤 | 1. 点击选择一个章节<br>2. 选择"详细"模式<br>3. 等待生成     |
| 期望 | 流式输出该章节详细展开内容，字数约 800-1500 字              |

**TC-READ-010：多章节选择与合并输出**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 大纲已展示                                                |
| 步骤 | 1. 通过复选框选择 2-3 个章节<br>2. 选择"缩略"或"详细"模式<br>3. 等待生成 |
| 期望 | 逐章生成内容，合并展示，各章以标题明确分隔                   |

**TC-READ-011：内容重新生成**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已生成某章节的缩略/详细内容                                |
| 步骤 | 1. 点击"重新生成"按钮                                      |
| 期望 | 重新调用 AI 生成不同版本内容，内容与前次有差异              |

### 2.4 知识心得

**TC-READ-012：知识心得 — 默认参数生成**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已选择一个章节                                            |
| 步骤 | 1. 选择"知识心得"模式<br>2. 保持默认风格和字数（300 字）<br>3. 等待生成 |
| 期望 | 生成笔记要点体的知识心得，字数接近 300 字                   |

**TC-READ-013：知识心得 — 切换预设风格**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已选择一个章节，处于"知识心得"模式                          |
| 步骤 | 1. 分别选择"学术严谨"和"幽默轻松"风格各生成一次             |
| 期望 | 两次输出的语气明显不同，"学术严谨"用词正式，"幽默轻松"风格活泼 |

**TC-READ-014：知识心得 — 自定义风格输入**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已选择一个章节，处于"知识心得"模式                          |
| 步骤 | 1. 在风格输入框输入自定义描述（如"像给小朋友讲故事一样"）<br>2. 生成 |
| 期望 | 生成内容的语气风格符合用户输入的描述                        |

**TC-READ-015：知识心得 — 自定义字数**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已选择一个章节，处于"知识心得"模式                          |
| 步骤 | 1. 将字数设为 1000 字<br>2. 生成                           |
| 期望 | 生成内容字数接近 1000 字（±20% 容差）                      |

**TC-READ-016：知识心得 — 字数校验**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 处于"知识心得"模式                                         |
| 步骤 | 1. 输入字数 50（低于 100）<br>2. 输入字数 5000（高于 3000） |
| 期望 | 显示校验提示"字数范围 100-3000"，不触发生成                 |

**TC-READ-017：知识心得 — 调整后重生成**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已生成一次知识心得                                         |
| 步骤 | 1. 修改风格为"实战导向"<br>2. 修改字数为 500<br>3. 点击重新生成 |
| 期望 | 按新参数重新生成内容，风格和字数符合新设定                   |

### 2.5 内容交互

**TC-READ-018：一键复制**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已生成内容                                                |
| 步骤 | 1. 点击"复制"按钮                                          |
| 期望 | 内容以 Markdown 格式复制到剪贴板，提示"已复制"              |

**TC-READ-019：流式输出体验**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 触发任意内容生成                                          |
| 步骤 | 1. 观察内容生成过程                                        |
| 期望 | 内容以打字机效果逐步展示，非等待完成后一次性显示            |

**TC-READ-020：加载状态与超时**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 触发搜索或生成                                            |
| 步骤 | 1. 观察加载过程                                            |
| 期望 | 展示加载动画/骨架屏；若超时（30s 无响应）提示"请重试"       |

### 2.6 数据持久化

**TC-READ-021：历史书籍列表**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已搜索并查看过至少一本书                                   |
| 步骤 | 1. 返回首页                                               |
| 期望 | 历史书籍列表展示之前搜索过的书籍，按最近访问排序            |

**TC-READ-022：生成内容自动保存**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 已生成某章节内容                                          |
| 步骤 | 1. 关闭页面<br>2. 重新打开该书同一章节                     |
| 期望 | 之前生成的内容可查看，无需重新生成                          |

**TC-READ-023：历史内容回看**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 同一章节已生成过多次内容（如不同模式/不同参数）              |
| 步骤 | 1. 进入该章节                                              |
| 期望 | 可查看该章节的历史生成记录列表，选择查看任一历史版本         |

### 2.7 响应式布局

**TC-READ-024：桌面端布局**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |

| 步骤 | 1. 打开书籍详情页                                          |
| 期望 | 左侧大纲 + 右侧内容双栏布局，信息密度合理                  |

**TC-READ-025：移动端布局**

| 项目 | 内容                                                     |
| ---- | -------------------------------------------------------- |
| 前置 | 浏览器宽度 ≤ 768px                                        |
| 步骤 | 1. 打开书籍详情页                                          |
| 期望 | 大纲和内容上下排列或抽屉切换，操作无遮挡、可正常点击        |
