# 执行产品经理完整流程

按 `.cursor/rules/product-requirement-flow.mdc` 中的规则执行产品经理完整流程。

## 执行要求

1. **先读规则**：读取并遵循 `product-requirement-flow.mdc` 的全部内容。
2. **先读技能**：读取 `skills/product-manager/SKILL.md` 中的「默认工作协议」与步骤。
3. **第一轮回复**：仅输出 **P0 澄清问题清单**（用户与场景、对象与范围、主路径、规则与边界、验收口径），不得跳过直接给方案或代码。
4. **等用户回答**：用户回复澄清答案后，再进入分析与方案阶段，产出 01-需求卡、02-需求方案、03-功能清单与验收标准；用户确认方案后，执行文档与可交互原型交付，并按规则完成自动化验收与问题交接。

## 触发方式

- 用户输入 `/pm-flow` 即表示要执行本流程。
- 若用户在同一消息中附带需求描述（如「一句话需求: xxx」），将其作为需求输入；否则从 P0 澄清问题清单开始。
