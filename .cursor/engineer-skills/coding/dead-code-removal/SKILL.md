# Dead Code Removal

安全识别并移除页面/组件中的无用代码，以及无人引用的组件文件和目录。

> **场景导航**
> - 本 skill 聚焦"移除"，不做逻辑重组。如需同步重构 → `/code-refactoring`
> - 涉及 React 行为变更风险 → `/react-guardrails`
> - 编码风格规范 → `/coding-standards`

## 硬约束

- **只删不改**：移除无用代码，不顺手重构逻辑、重命名、调整结构。
- **有疑必留**：对用途不确定的代码，保留并标注 `// TODO: 待确认是否可删`，不擅自删除。
- **不破坏运行时行为**：删除后页面所有功能、交互、请求、渲染结果必须与原来完全一致。
- **样式谨慎**：Less/CSS 类名可能被动态拼接（如模板字符串），删除前必须全局搜索确认无引用。

## 无用代码类型速查

| 类型 | 典型信号 | 风险等级 |
|------|----------|----------|
| 未使用的 import | TS/lint 报 `is declared but never used` | 低 |
| 未使用的局部变量 | 声明后从未读取 | 低 |
| 未使用的 useState | 声明了但 setter 和值都未在 JSX/逻辑中使用 | 中 |
| 永远为 false 的条件分支 | `if (false)`、常量条件 | 中 |
| 注释掉的旧代码块 | `// const xxx = ...`、`{/* <OldComp /> */}` | 低 |
| 未被调用的函数/handler | 函数定义但无调用点 | 中 |
| 未被渲染的组件导入 | import 了但 JSX 里没用 | 低 |
| 重复的 useEffect | 两个 effect 监听相同依赖做相同的事 | 高 |
| 覆盖前即被覆盖的赋值 | `let x = 1; x = 2;`（第一行无意义） | 中 |
| 无引用的 Less 类名 | `.old-card { ... }` 但 JSX 中没有对应 className | 中 |
| 无引用的组件文件 | 文件存在但全项目无 import | 高 |
| 废弃的目录 | 整个 `components/` 子目录无任何引用 | 高 |

## 执行流程

### 第一步：扫描识别

阅读目标文件，逐类扫描并列出候选项：

```
## 候选无用代码清单

### 未使用的 import
- [ ] line 3: `import { OldComponent } from "./components"`
- [ ] line 7: `import _ from "lodash"`（仅导入未使用）

### 未使用的状态/变量
- [ ] line 42: `const [oldFlag, setOldFlag] = useState(false)`
  - setter `setOldFlag` 无调用；`oldFlag` 无读取

### 未调用的函数
- [ ] line 88: `const handleOldAction = () => { ... }`
  - JSX 中无 `onClick={handleOldAction}` 等调用点

### 注释旧代码
- [ ] line 120-135: 注释掉的旧渲染逻辑

### 无引用的样式（需确认）
- [ ] `.old-section`：全局搜索 className 无匹配
```

### 第二步：逐项确认

对每个候选项，执行确认动作再删除：

| 类型 | 确认方式 |
|------|----------|
| import | 全文搜索标识符，确认零使用 |
| useState | 搜索变量名和 setter 名，确认 JSX 和逻辑中均无引用 |
| 函数/handler | 搜索函数名，确认无直接调用、无作为参数传递、无通过变量引用 |
| 条件分支 | 确认条件值是否有可能在运行时为 true（注意动态赋值） |
| 样式类名 | 全局搜索字符串（含动态拼接场景，如 `${prefix}-old`） |
| 注释代码 | 确认是旧逻辑残留而非"临时注释待恢复" |

**以下情况必须保留，不得删除：**
- 变量/函数通过字符串动态引用（`obj[key]`）
- 看似未使用但实际被子组件通过 `...props` 透传
- 注释掉的代码带有 `TODO`、`FIXME`、`TEMP` 等标记
- 样式类名在其他文件中使用（需跨文件搜索）

### 第三步：无用组件文件/目录扫描（跨文件级）

**适用场景**：用户明确要求清理无用文件，或在代码扫描中发现某个组件/文件已不再被引用。

**确认步骤：**

1. **定位候选文件**：在 `components/`、`hooks/`、`utils/` 等目录下找到疑似废弃的文件。
2. **全局搜索引用**：对每个候选文件，搜索其文件名（不含扩展名）和默认导出的组件/函数名，确认在整个 `src/` 范围内零引用。
3. **检查 index.ts 导出**：若文件通过 `components/index.ts` 导出，需同步移除该导出行。
4. **检查关联文件**：删除 `.tsx` 的同时删除同名 `.less`；删除整个子目录时确认目录内所有文件均无引用。

**必须保留的情况：**
- 文件被动态 import（`import('./Comp')`）
- 文件路径被字符串硬编码引用（如路由配置）
- 文件是某个 `index.ts` 的透传导出，且 `index.ts` 被外部引用

### 第四步：小步删除

每次只删除一类，删除后立即验证：

```
删除顺序（低风险 → 高风险）：
1. 注释掉的旧代码块
2. 未使用的 import
3. 未使用的局部变量/常量
4. 未调用的函数
5. 未使用的 useState（删前确认 setter 也无用）
6. 永远不执行的条件分支
7. 无引用的 Less 类名（需跨文件确认）
8. 无引用的组件文件/目录（需全局搜索确认）
```

### 第五步：验证

每删一批后检查：

- [ ] TypeScript 编译无新增错误
- [ ] ESLint 无新增警告
- [ ] 页面渲染结果与删除前一致
- [ ] 所有交互功能（点击、提交、导航）正常
- [ ] 网络请求未发生变化
